# Sealhouse Usage

Sealhouse is still early in its public fork lifecycle. Until a fuller user
manual is written, use the app around the core signing workflow:

1. Upload or create a document.
2. Add recipients or choose self-signing.
3. Place the required fields on the PDF.
4. Send the request or share the signing link manually.
5. Complete the signing flow.
6. Download the signed PDF and completion evidence.

For local development and smoke testing, use synthetic documents and generated
test identities only. Do not use real customer documents or signatures in public
examples, tests, screenshots, or CI artifacts.
